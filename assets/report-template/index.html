服务概述Service Overview

服务边界
执行规则

阿里云应急响应服务由经验丰富的安全专家为客户提供一对一专业服务，依托规范的服务流程与项目管理机制，协助客户在最短时间内、以最低成本解决紧急安全事件。

应急响应服务以远程事件分析为主，具体内容包括：排查主机是否被入侵、处置进行中的攻击、阻断攻击者进一步行动、分析攻击手法，并尽可能定位入侵原因。每一次服务都以"快速止损 + 持久建议"为目标。

交付物除本报告外，还包括补充的样本分析文档、IOC（失陷指标）清单，以及相关的遥测数据导出。

§ 2.3 · 事件结论
§ 2.3 · 结论

事件结论Incident Conclusion

置信度 · 高
Headline · 核心结论

外部攻击者利用一把泄露的管理员级 AccessKey，完成了对云租户的资产枚举，并在一台公网主机上部署了攻击工具，准备进一步横向移动。自告警触发起 6 小时内被完全抑制，在可观测范围内未发现数据外传迹象。

根本原因：某历史源代码仓库中遗留的凭证被公开采集。助推因素：AK 权限过宽、未配置 IP 白名单、缺少针对 AK 的异常检测策略。

事件定性
T1078.004 · 凭证泄露导致的未授权访问；攻击者完成租户范围资产枚举，并在一台公网主机部署工具集，尚未形成持久化据点。
影响范围
24 台在范围资产中 1 台被部署工具集；在可观测遥测范围内未发现数据外传、破坏或加密。
处置状态
已完成 — 泄露 AK 吊销、受影响主机下线重建、同组密钥轮转、出站策略加固。
残留风险
建议 7 日内扫描其他历史源码仓库以确认同类凭证未外泄；对 RAM 用户启用异常访问检测（UEBA）；按最小权限原则收敛 RAM 策略。

§ 三 · 技术分析
§ 三 · Three

技术分析Technical Analysis

手法 · 样本 · 路径 · ATT&CK

3.1 攻击手法 Attack Chain

本次入侵遵循典型的云原生攻击链：凭证泄露 → API 枚举 → 横向到单台主机 → 工具落地 → 持久化失败。攻击者首先通过公开渠道获取到历史仓库中遗留的管理员级 AccessKey（合法账户 T1078.004，AK 签发于 2023 年且从未轮转，权限范围过宽），随后使用该凭证从境外 IP 发起 API 侦察（云服务发现 T1580），集中调用 Bucket、实例与 RAM 用户相关的列表类接口，完成租户范围内的资产枚举。

在确认可达目标后，攻击者通过云侧 Run-Command 编排接口（T1651 类）向一台公网主机下发 Shell 指令（Unix Shell T1059.004），命令由对手的控制通道统一下发；主机拉起工具集后向两个主机托管商 IP 发起 HTTPS 回连（应用层协议 T1071.001），尝试建立反向隧道与网络扫描能力。完成抑制时攻击者尚未开展进一步权限提升，持久化尝试被出站阻断策略失败；在可观测遥测范围内未发现数据销毁、加密或确认的外传行为。

3.2 样本分析 Sample Analysis

§ 五 · 总结
§ 五 · Five

总结与安全建议Summary & Advisory

经验 · 现状 · 下一步

5.1 事件总结 Event Summary

机会型攻击者利用一把泄露的管理员凭证，对租户进行资产枚举、在一台主机落地工具，并在造成实质影响前被抑制。根本原因为管理员 AccessKey 泄露且未配置 IP 白名单；本次事件最大的短板集中在控制面侧的检测深度与 AK 生命周期管理，最大的强项则是告警触发后的宣布与抑制速度足够快。综合评估，当前残余风险为中等，待恢复阶段建议全部闭环后再行复评。

本次事件呈现出典型的现代云入侵特征：被攻破的不再是某台具体主机，而是控制面。一旦管理员级身份落入攻击者之手，其能力范围将覆盖该身份可触达的全部资源，本次即被机会式地选中了一台公网主机作为落脚点。

抑制之所以高效，关键在于告警来自云原生信号（异常地理位置 + 异常 API 组合），而非主机侧指标；若依赖主机侧告警，将过晚触发。这是本次事件最值得固化的经验。

下一步加固工作围绕三个主题展开：身份（退役管理员 AK、采用角色、保留的 AK 必须 IP 白名单化）、可观测性（集中化日志、控制面侧检测）、暴露面（减少对外暴露、建立服务基线）。

快捷指引 Cheatsheet

进入编辑

  • 右下角 ✎ 编辑 进入；✓ 完成 退出。编辑模式下文本被点到会高亮。

文字排版（系统快捷键）

  • 加粗 ⌘B · 斜体 ⌘I · 下划线 ⌘U（Windows 用 Ctrl）
  • 段内换行 Enter（仅段落/列表；其他格子 Enter 已屏蔽以防破坏格式）
  • 撤销 ⌘Z · 重做 ⌘⇧Z，也可用工具栏按钮

一处修改多处同步

  • 带橙色点状下划线的字段是同步锚点：改任一处，其他自动更新。
  • 同步范围：日期（封面 / 报告编号 / 所有页眉 / 尾页）、服务编号末段（01 / 02…）、版本号（封面 + 尾页，修订表不参与）

插入图片

  • 编辑模式下 ⌘V 粘贴剪贴板图片 · 或从 Finder 拖图到目标段落附近
  • 宽度超过 1600px 的图会自动压到 JPEG；单报告约可容纳 50 张
  • 鼠标移到图片，右上角浮出工具条：⅓ / ⅔ / ▭ 切换宽度，✕ 删除；在下一段按 Backspace 会选中图片，选中后再按 Delete / Backspace 删除

无序列表（项目符号）

  • 段首输入 - / * / + 加空格 → 自动转成 •（同 Markdown）
  • 在 • 行末按 Enter → 下一行继续新项；空 • 行再按 Enter → 退出列表，留在当前行写普通文字

小节与列表

  • 每页底部 + 小节 → 追加二级小节（3.1 / 3.2…），自动编号
  • 小节标题悬停出现 + 子 → 追加三级子小节（3.1.1 …）；✕ 删除整节（含子节内容）
  • 排查对象、时间线、行动项、参考建议、TOC 等列表都有 + / ✕

草稿自动保存

  • 编辑内容自动写入浏览器 IndexedDB；刷新后顶部横幅可选择 加载草稿 或 放弃草稿
  • 工具栏 ⟲ 放弃草稿 可手动清空当前草稿

不可编辑页

  • 03 目录：自动生成 · 04 服务概述：模板固定 · 14 版本与致谢：模板固定

导出

  • ⌘P 或 ⤓ 导出 PDF → 默认文件名 {客户}-安全事件应急响应报告-{日期}-{编号}.pdf
