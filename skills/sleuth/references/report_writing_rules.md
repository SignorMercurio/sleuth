# 报告填充细则与本项目特有约束（步骤 8 写作层）

SKILL.md 步骤 8 写报告前读取本文件。这里集中"模板逐块怎么填"与"本项目特有的报告约束"两类细则；句法/文风/反 AI 腔规则在 `references/report_style.md`，事实来源与禁止事项在 `references/findings_spec.md`「写作层使用规则」。三者配合使用。

## 模板逐块填充（对应 `assets/report.md`）

生成时先读取 `assets/report.md`，再编辑输出文件。保留 Markdown 标题、frontmatter 和 `:::` 指令块，只替换占位内容；模板的 HTML 写作指导注释原样保留（交付物是渲染后的 PDF，注释不可见）；不改写为 HTML，不添加 `<span style=...>`/`<font>`/内联 `color` 等视觉样式或字体/背景色。默认写"客户阅读版"短报告：结论前置，证据够支撑结论就停止展开，各节篇幅上限以模板注释为准；除非用户明确要求深度取证版，不新增长篇附录。

- **Frontmatter**：填写 `date`、`sir-seq`、`version`，模板自带的样值（如 `date: 20250507`）必须替换为本次事件值，不可原样保留；`client` 保持占位 `客户名称`，由交付人手工填写，AI 不改
- **封面块**：`::: cover-hero` 仅 `sev` 行按本次事件严重等级填写，其余是固定品牌信息保持原样；`::: cover-meta` 字段由 frontmatter 自动渲染，禁止手改——只需填好 frontmatter
- **修订记录**：`::: rev` 可按本次交付更新，描述字数与版本递增规则按模板注释执行
- **事件概述**：填写 `::: asset`、`::: timeline`、`::: callout`，以及事件定性、影响范围、处置状态、残留风险
- **技术分析**：填写排查过程、样本分析、入侵路径，正文只写支撑结论的证据链；`::: attack` 矩阵保持 12 战术×每战术 5 技术的固定版式，不增删/重排，只给 findings「ATT&CK 映射」一节列出的技术加 `!` 点亮，其余槽位留常见战术占位
- **响应行动与总结**：用 `[x]` / `[/]` / `[ ]` 区分已完成、进行中、未开始，只保留客户下一步必须知道的动作和与本次事件直接相关的参考资料
- **图片与元说明**：图片占位与图注格式按模板注释执行；日志覆盖缺口、调查范围等元说明用 `>` 引用块与证据叙事分层

## 本项目特有约束

- **结论可信度**：报告中标为「已确认/确定」的结论必须已通过步骤 7 的对抗式验证；未通过的按降级措辞写「推测/无法确认/未观察到」，不要保留确定措辞；不确定性措辞的具体写法（句中标注、双向校准）见 `references/report_style.md`
- **未决旁支线索的去向**：调查中没闭环、最终落到「需客户确认归属」的旁支（如某个存疑的登录来源、密钥指纹不一致、疑似第二入口），不写进客户报告正文，更不要跨章节反复出现——它们属于 findings 工作底稿。只有当它构成客户必须继续处理的残留风险时，才提炼成「残留风险」或「证据缺口」里的一条，其余删除
- **证据边界只完整展开一次**：同一条限定（如「请求与进程无法关联」「按密钥已泄露处置」）在结论 callout 或技术分析推导处选一个权威位置完整写出，其他章节需要时用短指针（「见核心结论」）或省略；交稿前按关键短语全文检索，同一限定出现 3 次以上必须收敛
- **范围声明一次、只说做了什么**：调查范围限制（快速溯源、按客户要求缩小范围）在排查过程用一句话声明；不逐项罗列未执行的手段（未查 ActionTrail、未启用 auditd 等），除非某项缺失本身构成客户必须处理的残留风险。「复评条件」类工作底稿内容不进客户报告
- **内部工作标识不进报告**：SIREN Client 编号、findings 底稿文件名等内部标识不得出现；云资源 ID（镜像 `m-*`、实例 `i-*`）默认省略，仅当客户需要据此定位实例执行动作时保留（如在产实例复核项）；资产以实例名 + IP 标识
- **历史独立事件不占版面**：与本次入侵无手法或基础设施关联的历史事件（如数年前的挖矿）不进时间线、ATT&CK 说明和响应行动；必须提及时在资产用途或技术分析一句话带过
- **云端证据写法**：SLS/SAS 类证据必须写明日志源、时间窗口、最早/最新记录与覆盖限制。主机侧报告至少覆盖网络外联、进程启动、远程/数据库登录、SAS 告警四类；进程侧无命令执行证据时也要写明进程日志最早覆盖时间，避免把「未检出」误写成「未发生」
- **文风**：执行 `references/report_style.md`，命中其反 AI 腔清单的句子出现即改写
- **IoC 展示层转义**：报告可见文本里攻击者侧 IoC 的 IPv4 地址和域名只把最后一个点替换成 `[.]`（如 `1.1.1.1` → `1.1.1[.]1`、`evil.example[.]com`），URL 协议头写成 `hxxp://`/`hxxps://`；受害资产自身的 IP/域名不转义。命令执行、检索过滤、内部分析仍用原始值
- **命令证据**：只把能支撑结论的关键命令和最短输出片段放进对应章节；不要把命令流水账、完整日志或低价值排查过程搬进报告
- **详略边界**：样本分析、ATT&CK 覆盖说明、参考资料都服务于本事件结论；没有样本或无关资料时用一句话说明或删除，不用模板内容凑篇幅
