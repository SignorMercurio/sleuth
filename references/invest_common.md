# 通用调查技巧

## 时间线重建
1. 收集所有时间戳信息
2. 按时间顺序排列事件
3. 关联不同数据源的事件
4. 识别攻击的关键节点

## 证据链构建
1. 从告警出发，向前追溯入侵源头
2. 从入侵点出发，向后追踪攻击行为
3. 关联所有相关证据
4. 验证攻击路径的合理性

## IoC 提取规范
- **IP 地址**: 攻击源 IP、C2 服务器 IP
- **域名**: 恶意域名、C2 域名
- **URL**: 恶意链接、下载源
- **文件路径**: 恶意文件的完整路径
- **文件哈希**: MD5、SHA1、SHA256
- **进程名**: 恶意进程名称
- **端口号**: 异常通信端口
- **用户名**: 被利用或创建的账户

## 限制输出技巧
- 使用 `tail -n N` 或 `head -n N` 限制行数
- 使用 `grep "<时间>"` 过滤时间范围
- 使用 `grep "<IP>"` 过滤特定 IP
- 使用 `awk`、`sed` 提取关键字段
- 先用 `wc -l` 评估输出大小

## 互联网资源利用
- 使用联网检索工具查询 CVE 详情
- 查询 IP 归属地和信誉
- 查询文件哈希（VirusTotal）
- 查询 ATT&CK 技术详情
- 查询威胁情报数据库
