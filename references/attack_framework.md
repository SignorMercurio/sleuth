# MITRE ATT&CK 框架参考

用于应急响应报告中的攻击行为映射：战术 → 技术（ID + 名称）+ 子技术编号。编号格式 `T[数字]` 或 `T[数字].[子编号]`。映射时优先用具体子技术，每条映射必须有溯源证据支撑，未涉及的战术省略；同一行为可能跨多个战术（如定时任务可属执行/持久化/权限提升）。

> **战术名 ↔ 报告矩阵对照**：报告 `::: attack` 矩阵用固定中文战术名，与下文 ATT&CK 标准译名略有出入，映射时对号入座（矩阵名不可改）：执行→执行攻击、持久化→持久控制、凭证访问→凭据访问、**发现→环境观察**、收集→收集信息、命令与控制→命令控制、**数据外传→窃取数据**、影响→侵害影响；初始访问 / 权限提升 / 防御规避 / 横向移动 两边一致。
